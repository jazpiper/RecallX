## Workflow

- Memforge is the primary durable memory system in this workspace. Use Linear for execution tracking and Notion for human-readable docs when available.
- Before meaningful work, check Memforge for relevant context using project names, features, issue IDs, file names, or key concepts. If a node is already known, prefer `memforge_context_bundle`; otherwise start with `memforge_search_nodes`.
- Read relevant memory before making assumptions. If Linear or Notion context is likely to matter, check those too.
- During work, record durable facts in Memforge: decisions, stable facts, open questions, useful summaries, and relationships between projects, issues, notes, and artifacts.
- After meaningful work, write back what changed, why it changed, and any follow-ups. Reflect important outcomes in Linear or Notion too when appropriate.
- If no relevant Memforge, Linear, or Notion context exists, say so briefly and continue.
- Keep memory concise, reusable, and searchable. Prefer canonical nodes for durable concepts and suggested relations until reviewed.
- Never store or expose secrets, credentials, tokens, or other sensitive data.
